## 一、产品概述

### 1.1 产品简介

加单商品分配系统是一个跨平台的智能补货分配工具，支持Windows、macOS和Linux系统。系统基于库存数据、销售数据和卖场等级，按照四阶段算法智能分配加单商品，确保高等级卖场、高销尽率卖场获得优先供应。

### 1.2 核心特性

1. **四阶段智能分配**：断码修复 → 销量匹配 → 销尽率优先 → 剩余分配
2. **可配置参数**：覆盖周期、等级权重、安全系数、最小目标库存、阶段优先级
3. **跨平台支持**：Windows、macOS、Linux
4. **用户友好界面**：现代化极简设计，支持阶段顺序调整
5. **分配原因记录**：详细记录每个卖场每个SKU的分配原因

### 1.3 应用场景

- 零售企业补货分配
- 服饰商品断码修复
- 按销售表现分配库存
- 多门店分级管理库存

## 三、四阶段分配算法

### 3.1 算法流程图

```
┌─────────────────────────────────────────────────────────┐
│                    加单商品分配流程                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  阶段1: 断码修复（优先级可配置）                          │
│  ├─ SA/A级：核心尺码≥2件，非核心尺码≥1件                 │
│  └─ 其他等级：核心尺码≥1件                               │
│                                                          │
│                          ↓                               │
│                                                          │
│  阶段2: 销量匹配（基于供应链公式，优先级可配置）          │
│  ├─ 目标库存 = 平均日需求 × 覆盖周期 + 安全库存         │
│  ├─ 覆盖周期：SA(30) > A(30) > B(14) > C(14) > D/OL(14) │
│  └─ 安全系数：SA(0.5) > A(0.4) > B(0.3) > C(0.25) > ... │
│                                                          │
│                          ↓                               │
│                                                          │
│  阶段3: 销尽率优先分配（所有等级参与，优先级可配置）      │
│  ├─ 综合得分 = 销尽率 × 等级权重                        │
│  ├─ 等级权重：SA(1.5) > A(1.3) > B(1.2) > C(1.1) > ... │
│  └─ 分配上限 = 销量 × 等级权重                          │
│                                                          │
│                          ↓                               │
│                                                          │
│  阶段4: 剩余分配（按等级优先级，固定最后执行）            │
│  ├─ 优先级：SA > A > B > C > D > OL                     │
│  └─ 单卖场上限：10件                                    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 各阶段详细说明

#### 阶段1：断码修复

**目的**：确保每个卖场达到最低库存要求，避免核心尺码断码

**规则**：
- SA/A级卖场：核心尺码（160/165）至少2件，非核心尺码至少1件
- B/C/D/OL级卖场：核心尺码至少1件

**计算公式**：
```
分配数量 = max(0, 目标库存 - 当前库存)
```

**触发场景**：新品上市、季中补货、断码严重的SKU

---

#### 阶段2：销量匹配

**目的**：根据历史销量和卖场等级，分配科学合理的库存

**供应链公式**：
```
平均日需求 = 30天销量 ÷ 30
安全库存 = 平均日需求 × 安全系数 × 覆盖周期
目标库存 = 平均日需求 × 覆盖周期 + 安全库存
简化为：目标库存 = 30天销量 ÷ 30 × 覆盖周期 × (1 + 安全系数)
```

**参数配置**：
- 覆盖周期（天）：按卖场等级设置
- 安全系数：按卖场等级设置
- 最小目标库存：确保销量为0时仍有基础库存

**示例计算**：
- SA级卖场，30天销量60件：
  - 目标库存 = 60 ÷ 30 × 30 × (1 + 0.5) = 2 × 30 × 1.5 = 90件
- B级卖场，30天销量30件：
  - 目标库存 = 30 ÷ 30 × 14 × (1 + 0.3) = 1 × 14 × 1.3 = 18件

---

#### 阶段3：销尽率优先分配

**目的**：优先分配给销尽率高且等级高的卖场，加快周转

**计算公式**：
```
销尽率 = 30天销量 ÷ (30天销量 + 当前库存)
综合得分 = 销尽率 × 等级权重
分配上限 = max(30天销量 × 等级权重, 2)
```

**排序规则**：
1. 所有等级卖场都参与（SA/A/B/C/D/OL）
2. 按综合得分从高到低排序
3. 得分相同的按等级顺序排序

**示例说明**：
- SA级卖场，销尽率50%：综合得分 = 0.5 × 1.5 = 0.75
- B级卖场，销尽率70%：综合得分 = 0.7 × 1.2 = 0.84
- 结果：B级卖场综合得分更高，优先获得分配

---

#### 阶段4：剩余分配

**目的**：将剩余库存按等级优先级分配完毕

**规则**：
1. 按等级顺序分配：SA → A → B → C → D → OL
2. 同等级内按顺序依次分配
3. 单卖场上限10件（超过则分配到上限后跳过）

**终止条件**：所有加单数量分配完毕或所有卖场都达到上限

## 四、数据格式要求

### 4.1 必需Excel表结构

系统要求输入的Excel文件包含以下4个工作表：

#### 1. 库存表（sheet名：库存）

| 列名 | 数据类型 | 说明 |
|------|---------|------|
| 卖场代码 | 文本 | 卖场唯一标识，需与卖场等级表一致 |
| 条码 | 文本 | SKU唯一标识，需与加单数量表一致 |
| 库存数量 | 整数 | 当前库存数量（≥0） |

#### 2. 销售表（sheet名：销售）

| 列名 | 数据类型 | 说明 |
|------|---------|------|
| 条码.条码 | 文本 | SKU唯一标识 |
| 店仓.卖场代码 | 文本 | 卖场唯一标识 |
| 数量 | 整数 | 近30天销售数量（≥0） |

#### 3. 卖场等级表（sheet名：卖场等级）

| 列名 | 数据类型 | 说明 |
|------|---------|------|
| 代码 | 文本 | 卖场唯一标识 |
| 卖场等级 | 文本 | 卖场等级，取值：SA/A/B/C/D/OL |

#### 4. 加单数量表（sheet名：加单数量）

| 列名 | 数据类型 | 说明 |
|------|---------|------|
| SKU | 文本 | SKU唯一标识 |
| SKC | 文本 | SKC分类标识（系统暂不使用，但必须有） |
| 需分配数量 | 整数 | 需要分配的总数量（≥1） |

### 5.2 功能区详细说明

#### 5.2.1 参数配置区

**位置**：界面顶部，可折叠区域

**功能**：
- 覆盖周期配置：按SA/A/B/C/D/OL等级配置（单位：天）
- 等级权重配置：按等级配置权重系数
- 安全系数配置：按等级配置安全库存系数
- 最小目标库存：按等级配置最小库存保障

**交互逻辑**：
1. 点击标题栏的 [▼] 可折叠/展开配置区域
2. 每个输入框支持输入数字，输入框为蓝灰色背景，黑色字体
3. 点击 [恢复默认] 按钮，所有参数恢复到默认值
4. 点击 [保存配置] 按钮，保存配置到 allocation_config.json

**参数默认值**：
| 参数 | SA | A | B | C | D | OL |
|------|----|---|---|---|---|----|
| 覆盖周期 | 30 | 30 | 14 | 14 | 14 | 14 |
| 等级权重 | 1.5 | 1.3 | 1.2 | 1.1 | 1.1 | 1.0 |

#### 5.2.2 分配逻辑说明区

**位置**：参数配置区下方，可折叠区域

**功能**：
- 阶段顺序调整：通过3个下拉框选择每个阶段的内容
- 阶段展示：以彩色卡片形式展示4个阶段的说明

**阶段选项**：
| 阶段ID | 阶段名称 | 说明 |
|--------|---------|------|
| broken_size_fix | 断码修复 | 优先填充缺码关键SKU |
| sales_match | 销量匹配 | 依据历史销速加权分配 |
| sell_through_priority | 销尽率优先 | 高销尽门店获得补货权重 |

**注意**：第4阶段"剩余分配"固定在最后执行，不可调整顺序

**交互逻辑**：
1. 点击标题栏的 [▼] 可折叠/展开逻辑说明区域
2. 每个下拉框包含3个阶段名称供选择
3. 点击 [应用顺序] 按钮，检查是否有重复选择，确认后更新阶段顺序
4. 点击 [恢复默认] 按钮，阶段顺序恢复到默认（1-断码修复，2-销量匹配，3-销尽率优先）
5. 下方彩色卡片实时显示当前阶段顺序

**颜色主题**：

#### 5.2.3 文件上传区

**位置**：逻辑说明区下方

**功能**：
- 文件选择：点击上传区域选择Excel文件
- 执行分配：点击按钮运行四阶段分配算法
- 保存结果：分配完成后保存到新Excel

**交互逻辑**：

**文件选择流程**：
1. 初始状态："执行加单分配"按钮为灰色禁用状态
2. 点击上传区域（虚线框），弹出文件选择对话框
3. 选择文件后，上传区域显示选择的文件名
4. "执行加单分配"按钮变为蓝色可点击状态，悬停时加深

**执行分配流程**：
1. 点击"执行加单分配"按钮
2. 状态区更新为"处理中..."，颜色为橙色（#D97706）
3. 系统读取Excel文件的4个工作表
4. 执行四阶段分配算法
5. 分配完成后，状态区更新为"分配完成"，颜色为绿色（#059669）
6. 结果预览区显示前20行分配结果
7. "导出清单"按钮变为可点击状态
8. 弹出提示框显示"加单分配完成"

**异常处理**：
- Excel文件格式错误：弹出错误提示框，显示错误信息
- 数据验证失败：显示具体错误信息
- 分配过程异常：捕获并显示错误详情

**保存结果流程**：
1. 点击"导出清单"按钮
2. 弹出文件保存对话框
3. 选择保存位置和文件名
4. 保存包含2个工作表的Excel文件：
   - 工作表1：分配数量（每个卖场每个SKU的分配数量）
   - 工作表2：分配原因（每个分配的原因说明 + 卖场等级）
5. 弹出提示框显示"结果已保存到: [文件路径]"

## 九、常见问题

### Q1: 程序打开后提示"初始化失败"？

A: 请检查Python环境是否完整，确保安装了pandas、openpyxl等依赖库。如果使用打包版本，请确保下载了对应平台的可执行文件。

### Q2: 执行分配时提示"找不到工作表"？

A: 请检查Excel文件名是否包含必需的4个工作表：库存、销售、卖场等级、加单数量。表名需与文档完全一致。

### Q3: 如何调整阶段优先级？

A: 在"分配逻辑说明"区域，通过三个下拉框分别选择第1-3阶段的内容，然后点击"应用顺序"按钮即可。第4阶段"剩余分配"固定在最后。

### Q4: 配置参数保存后，重新打开程序会恢复默认吗？

A: 不会。配置会保存到allocation_config.json文件，下次启动会自动加载。如需恢复默认，可点击界面上的"恢复默认"按钮。

### Q5: 分配数量与加单数量不一致怎么办？

A: 这种情况通常是因为所有卖场都已达到上限。检查分配原因表，查看是否大量记录为"剩余分配"且达到上限。

### Q6: Mac上提示文件损坏或无法打开？

A: 需要在系统设置 - 安全性与隐私中允许运行来自任何来源的应用，或右键点击选择"打开"。

## 十、附录

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| SKU | Stock Keeping Unit，库存量单位，单个商品规格 |
| SKC | Stock Keeping Color，通常指同一款式不同颜色 |
| 销尽率 | 销量 ÷ (销量 + 库存)，表示库存周转快慢 |
| 覆盖周期 | 库存可以支撑的销售天数 |
| 安全库存 | 为防止缺货而准备的缓冲库存 |
| 核心尺码 | 销售最好的尺码，如服饰类的160、165 |
| 卖场等级 | SA(旗舰店) > A(标杆店) > B(标准店) > C(一般店) > D(低销店) > OL(线上) |

### 10.2 分配原因枚举

| 原因格式 | 说明 |
|---------|------|
| 断码修复(N) | 为确保断码SKU有基础库存，N为本次分配数量 |
| 销量匹配(N) | 根据销量和覆盖周期计算的分配，N为本次分配数量 |
| 销尽率优先(N) | 根据综合得分排序分配，N为本次分配数量 |
| 剩余分配(N) | 按等级顺序分配的剩余库存，N为本次分配数量 |
