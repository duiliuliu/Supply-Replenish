# 加单商品分配系统

## 项目概述

加单商品分配系统是一个跨平台的工具，用于根据库存、销售数据和卖场等级，智能分配加单商品到各个卖场。系统支持Windows、macOS和Linux平台，提供直观的图形界面，方便用户操作。

## 数据格式要求

系统需要读取包含以下工作表的Excel文件：

### 1. 库存表 (库存)
- **字段要求**：
  - `卖场代码`：卖场的唯一标识
  - `条码`：商品的唯一标识
  - `库存数量`：当前库存数量

### 2. 销售表 (销售)
- **字段要求**：
  - `条码.条码`：商品的唯一标识
  - `店仓.卖场代码`：卖场的唯一标识
  - `数量`：销售数量

### 3. 卖场等级表 (卖场等级)
- **字段要求**：
  - `代码`：卖场的唯一标识
  - `卖场等级`：卖场等级，支持的值包括：SA、A、B、C、D、OL

### 4. 加单数量表 (加单数量)
- **字段要求**：
  - `SKU`：商品的唯一标识
  - `SKC`：商品分类标识
  - `需分配数量`：需要分配的商品数量

## 加单分配逻辑

系统采用以下分配策略，按优先级执行：

### 1. 断码修复
- **优先级**：最高
- **逻辑**：
  - SA/A级卖场：核心尺码(160, 165)至少保证2件，非核心尺码至少保证1件
  - 其他等级卖场：核心尺码至少保证1件
  - 确保每个卖场的库存达到最低要求

### 2. 销量匹配
- **优先级**：第二
- **逻辑**：
  - 根据30天销量，为每个卖场分配相应数量的商品
  - 确保库存与销量相匹配

### 3. B/C/D/OL级按销尽率降序分配
- **优先级**：第三
- **逻辑**：
  - 对B/C/D/OL级卖场，按销尽率（销量/总库存）降序排序
  - 优先分配给销尽率高的卖场

### 4. 剩余分配
- **优先级**：最低
- **逻辑**：
  - 将剩余商品分配给所有卖场
  - 确保每个卖场的单个尺码库存不超过15件

## 验证逻辑

系统在执行分配前会进行以下验证：
- 检查Excel文件是否包含所有必要的工作表
- 检查工作表是否包含必要的字段
- 验证数据格式是否正确
- 确保所有数值字段为非负整数

## 输出结果

系统生成的分配结果包含两个部分：

### 1. 分配数量
- 行：每个卖场
- 列：每个需要分配的SKU
- 值：分配给该卖场的该SKU数量

### 2. 分配原因
- 行：每个卖场
- 列：每个需要分配的SKU
- 值：该分配的原因

用户可以将结果保存为新的Excel文件，包含两个工作表：
- `分配数量`：包含分配的具体数量
- `分配原因`：包含每个分配的原因

## 分配流程说明

分配流程按照以下优先级顺序执行，只要还有剩余待分配数量，就会继续下一阶段：

1. **断码修复** → **销量匹配** → **销尽率优先** → **剩余分配**

**是的，即使在断码修复分配了货品数量后，如果还有剩余需要分配的数量，就会继续进行销量匹配、销尽率优先、剩余分配等后续步骤。

## 分配原因枚举

| 分配原因 | 说明 |
| --- | --- |
| 断码修复(N) | 为了确保卖场的库存达到最低要求，SA/A级卖场核心尺码至少保证2件，非核心尺码至少保证1件；其他等级卖场核心尺码至少保证1件。N为本次分配数量。 |
| 销量匹配(N) | 根据30天销量，为每个卖场分配相应数量的商品，确保库存与销量相匹配。N为本次分配数量。 |
| 销尽率优先(N) | 对B/C/D/OL级卖场，按销尽率（销量/总库存）降序排序，优先分配给销尽率高的卖场。N为本次分配数量。 |
| 剩余分配(N) | 将剩余商品分配给所有卖场，确保每个卖场的单个尺码库存不超过15件。N为本次分配数量。 |

## 使用说明

### 运行系统
1. 下载对应平台的可执行文件：
   - Windows：`加单分配系统_v3_Windows.exe`
   - macOS：`加单分配系统_v3_Mac.app`
   - Linux：`加单分配系统_v3_Linux`

2. 启动系统后，点击"浏览"按钮选择包含所需数据的Excel文件
3. 点击"执行加单分配"按钮开始分配过程
4. 查看分配结果预览（前20行）
5. 点击"保存结果"按钮将分配结果保存为新的Excel文件
