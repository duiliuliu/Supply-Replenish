# 加单商品分配 v3 - 跨平台GUI程序

## 简介
本版本提供完整的图形界面程序，支持在Windows、Mac和Linux平台上运行，无需安装Excel或Python环境。

## 文件说明

| 文件 | 说明 |
| --- | --- |
| 💡加单商品分配_0424.xlsx | 原始Excel文件 |
| allocation_core.py | 加单分配核心逻辑 |
| allocation_app.py | GUI主程序 |
| requirements.txt | Python依赖包列表 |
| pyinstaller_build.py | 打包脚本 |
| 使用说明_v3.md | 本使用说明文档 |

## 方法一：直接运行Python程序（需要Python环境）

### 安装环境
1. 确保已安装 Python 3.7 或更高版本
2. 在终端或命令行中，进入v3目录
3. 安装依赖：
```bash
pip install -r requirements.txt
```

### 运行程序
```bash
python allocation_app.py
```

### 使用流程
1. 程序启动后，点击「浏览」按钮选择Excel文件
2. 选择文件后，点击「执行加单分配」
3. 等待分配完成（会显示进度状态）
4. 分配完成后可以预览前20行
5. 点击「保存结果」按钮，选择保存位置

## 方法二：打包成可执行文件（推荐，无需Python环境）

### 打包前准备
1. 确保已安装 Python 3.7+
2. 安装依赖包：
```bash
pip install -r requirements.txt
pip install pyinstaller
```

### Windows系统打包
```bash
python pyinstaller_build.py
```
或者直接执行：
```bash
pyinstaller --onefile --windowed --name "加单分配系统_v3" allocation_app.py --add-data "allocation_core.py;."
```

### Mac系统打包
```bash
python pyinstaller_build.py
```
或者直接执行：
```bash
pyinstaller --onefile --windowed --name "加单分配系统_v3" allocation_app.py --add-data "allocation_core.py:."
```

### Linux系统打包
```bash
python pyinstaller_build.py
```
或者直接执行：
```bash
pyinstaller --onefile --windowed --name "加单分配系统_v3" allocation_app.py --add-data "allocation_core.py:."
```

打包完成后，可执行文件位于 `dist` 目录下。

## 使用方法说明

### 主界面功能
1. **文件选择**：点击「浏览」选择Excel文件
2. **执行分配**：点击「执行加单分配」运行分配逻辑
3. **保存结果**：分配完成后可保存为新Excel
4. **结果预览**：自动显示前20行分配结果
5. **状态提示**：实时显示当前运行状态

### 分配逻辑说明
程序内部执行与v1完全相同的分配步骤：
1. 在库存表计算SKC列（LEFT(K列,12)）
2. 按SA→A→B→C→D→OL排序
3. 断码修复 → 销量匹配 → 销尽率分配 → 剩余分配
4. 单SKU单卖场≤15限制

### 验证方法
使用v1的验证逻辑或v2的Excel宏进行验证。

## 注意事项

1. **Excel文件格式**：确保Excel包含所需的表（库存、销售、卖场等级、加单数量）
2. **数据完整性**：确保数据格式正确，特别是SKU和卖场代码列
3. **保存格式**：保存结果时选择.xlsx格式
4. **平台兼容性**：打包后的程序只能在对应系统上运行，Windows打包的只能在Windows运行，Mac同理

## 常见问题

### Q: 程序打开后无法选择文件？
A: 确保您有文件选择权限，并且文件格式正确（.xlsx或.xlsm）

### Q: 执行分配时提示错误？
A: 请检查Excel文件中是否包含所有必需的表，表名和列名需与原始文件完全一致。

### Q: 打包后文件很大？
A: PyInstaller打包会包含所有依赖库，这是正常现象。可以通过调整打包参数优化大小。

### Q: Mac上提示文件损坏或无法打开？
A: 需要在系统设置 - 安全性与隐私中允许运行来自任何来源的应用。
